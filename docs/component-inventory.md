## Blazor UI components

### Party Picker (`Hexalith.Parties.Picker`)

Self-contained RCL (no FluentUI dependency). Searches via `IPartiesQueryClient.SearchPartiesAsync`; emits a **durable party-id** selection (Blazor `EventCallback` + DOM `party-selected` CustomEvent for the custom element). Embeds as `<PartyPicker>` in Blazor or `<hexalith-party-picker>` in any JS app. The control uses Fluent 2 token aliases when hosted in the FrontComposer shell and exposes input-owned WAI-ARIA combobox semantics (`aria-activedescendant`, listbox/options, keyboard selection). Full integration guide: **[frontend/party-picker.md](frontend/party-picker.md)**.

| Component | Category | Key parameters |
|-----------|----------|----------------|
| `PartyPicker.razor` | Form / Input (embeddable) | `SelectedPartyId`(+Changed), `AccessTokenProvider`/`AccessToken`/`RequestCustomizer`, `ContextKey`, `AuthContextKey`, `SearchMode`, `PageSize`, `Disabled`, `ReadOnly`, `Labels`, `ResultTemplate`, `DispatchDomEvents` |

Supporting services: `PartyPickerApiClient` (the only bridge to `IPartiesQueryClient`), `PartyPickerLabels` (~50 localizable strings), `PartyPickerDefaults` (tag `hexalith-party-picker`, event `party-selected`, pageSize 10, debounce 300ms), state enums.

### Admin Portal (`Hexalith.Parties.AdminPortal`)

FrontComposer-hosted **FluentUI** RCL (render mode owned by the host shell). Routed at `/admin/parties`, `/admin/parties/{id}`, `/admin/parties/{id}/gdpr`, `/admin/parties/new`, and `/admin/parties/{id}/edit`. Talks to the backend via `IPartiesQueryClient` (preferred) or the FrontComposer `IQueryService`, `IPartiesCommandClient` through the AdminPortal adapter for create/edit, and `IAdminPortalGdprClient` for GDPR. Admin functions: tenant-scoped browse (debounced server search, filters, paging), responsive party detail, create/edit with validation, related-party picker binding, EventStore-admin deep links, and GDPR surfaces for erasure, restriction, consent, portability export, processing records, erasure certificate display, and verification retry.

### Consumer Portal (`Hexalith.Parties.ConsumerPortal`)

FrontComposer-hosted **FluentUI** Consumer RCL. Routed at `/me`, `/me/edit`, `/me/consent`, and `/me/privacy`; every route carries `Authorize(Policy = "Consumer")`. The implemented Epic 4 surfaces are read-only My Profile and Edit My Profile. Consent and privacy routes are protected shells for Epic 5.

ConsumerPortal does not reference `Hexalith.Parties.UI` and does not use gateway clients directly. It owns narrow caller-id-free ports:

- `IConsumerProfileDataClient.GetMyPartyAsync(...)`
- `IConsumerProfileEditClient.UpdateMyProfileAsync(...)`

The `parties-ui` host registers scoped adapters for those ports and delegates to `ISelfScopedPartiesClient`, which injects the resolved `party_id` and forbids list/search or arbitrary party ids.

| Component | Category | Purpose |
|-----------|----------|---------|
| `MyProfilePage.razor` | Page (routable) | self-profile read, freshness cue, stale/degraded handling, erased tombstone |
| `EditMyProfilePage.razor` | Page (routable) | self-profile edit for Person/Organization details with validation and one polite save status source |
| `FreshnessStatus.razor` | Display | dot plus text freshness indicator with polite live-region semantics |
| `ProfileField.razor` | Display | reusable profile label/value rendering |
