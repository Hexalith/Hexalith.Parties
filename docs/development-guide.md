# Development Guide — Hexalith.Parties

> Generated by BMAD *Document Project* (deep scan, 2026-06-02). The canonical onboarding walkthrough (with auth, first command/query, MCP) is **[getting-started.md](getting-started.md)** — this guide is the build/test/convention reference.

## Prerequisites

| Tool | Version |
|------|---------|
| .NET SDK | **10.0.300+** (pinned in `global.json`, rollForward latestPatch) |
| Docker Desktop | latest (must be running before Aspire) |
| Git | recent |
| `jq` | optional (Bash examples) |
| PowerShell `pwsh` 7+, `kubectl`, DAPR CLI, `aspirate` 9.1.0 | only for the optional Kubernetes path (`aspirate` is restored by `dotnet tool restore`) |

## 1. Clone & submodules

The build resolves `Hexalith.EventStore` and `Hexalith.Tenants` as project references. They are **not** checked out in a fresh clone — initialise the root-level submodules:

```bash
git clone https://github.com/Hexalith/Hexalith.Parties.git
cd Hexalith.Parties
git submodule update --init Hexalith.EventStore Hexalith.Tenants
# Do NOT use --recursive for the default local run (the build gate forbids nested-submodule init).
```

Optional rich search adds the `Hexalith.Memories` submodule (run with `EnableMemoriesSearch=true`).

## 2. Build

```bash
dotnet restore Hexalith.Parties.slnx
dotnet build Hexalith.Parties.slnx --configuration Release --no-restore
```

`TreatWarningsAsErrors=true` is solution-wide. A green build on a fresh clone (root submodules only, no warnings override) is the **build gate** — verify parity locally with:

```bash
bash scripts/check-no-warning-override.sh
```

See [build-gate.md](build-gate.md) for the policy and escape valves (narrow `<NoWarn>RuleId</NoWarn>` in the consuming csproj with an issue link — never a command-line `TreatWarningsAsErrors=false`).

## 3. Run locally (one command)

```bash
dotnet aspire run --project src/Hexalith.Parties.AppHost
```

Open the Aspire dashboard URL and confirm `eventstore`, `eventstore-admin`, `parties`, `tenants`, `redis`, the DAPR sidecars, `statestore`, and `pubsub` are healthy. `eventstore-admin-ui` and `parties-mcp` are **explicit-start** — start them from the dashboard when needed. AppHost http profile: `http://localhost:15100` (OTLP `21100`).

Treat the system as usable only after `eventstore`, `parties`, and `tenants` are healthy (public traffic enters through EventStore). Health endpoints per service: `/ready`, `/health`, `/alive`.

### AppHost feature flags (orchestration-time)

| Flag | Effect |
|------|--------|
| `EnableMemoriesSearch=true` | composes the `memories` service + injects `Parties__MemoriesSearch__Enabled=true` (requires the Memories submodule) |
| `EnableEventStoreSampleUi=true` | composes `sample` + `sample-blazor-ui` |
| `EnableKeycloak` | local Keycloak IdP (default true in run mode; publish uses the external `tache` realm) |
| `PUBLISH_TARGET` | `docker` / `k8s` / `aca` for Aspire-native publish (orthogonal to the aspirate deploy path) |

## 4. Test

```bash
dotnet test Hexalith.Parties.slnx --configuration Release        # everything
dotnet test tests/Hexalith.Parties.Server.Tests/Hexalith.Parties.Server.Tests.csproj   # one project
```

Or use the **lane runner** (`scripts/test.ps1 -Lane <lane>`, default `unit`, Release):

| Lane | Scope |
|------|-------|
| `unit` | Contracts, Client, Server, Projections, Security, AdminPortal, Picker, Mcp |
| `integration` | Hexalith.Parties.Tests + Sample.Tests |
| `topology` | Hexalith.Parties.IntegrationTests (full Aspire topology — needs Docker/DAPR; skips gracefully if absent) |
| `deploy` | Hexalith.Parties.DeployValidation.Tests (static `deploy/` validation) |
| `all` | whole solution |
| `coverage` | whole solution with `--collect "XPlat Code Coverage"` |

**Frameworks:** xUnit v3 + Shouldly (assertions) + NSubstitute (mocking); bunit for Blazor components; Testcontainers + `Aspire.Hosting.Testing` for integration; YamlDotNet for manifest validation. See [component-inventory.md](component-inventory.md) for per-project coverage.

## 5. Configuration & feature flags (runtime)

Read by the `parties` host (`appsettings*.json` / env, `__` nesting):

| Section / key | Purpose |
|---------------|---------|
| `Authentication:JwtBearer` | JWT validation (OIDC `Authority` or symmetric `SigningKey` ≥32 chars; `Issuer`, `Audience`) |
| `Tenants` | tenant integration (ServiceName, PubSubName, TopicName `system.tenants.events`, CommandApiAppId) |
| `Parties:Projections` | `ProjectionOptions` (BatchSize > 0, BatchTimeWindowMs > 0) |
| `Parties:CryptoShredding:IsEnabled` | crypto-shredding feature (**default true**) — see GDPR note below |
| `Parties:Compliance:GdprFeaturesActive` | suppresses the MVP compliance warning (**default false**) |
| `Parties:MemoriesSearch` | rich search (Enabled, Endpoint, ApiToken, TenantId, CaseId, EnabledAxes) |
| `EventStore:CommandStatus` / `EventStore:DomainServices` | EventStore server integration |

> ⚠️ **GDPR toggles are independent.** `Parties:CryptoShredding:IsEnabled` (the feature) defaults **on**; `Parties:Compliance:GdprFeaturesActive` (the MVP warning) defaults **off**. The README's "GDPR not in MVP" notice refers to the *warning*, not the encryption feature, which is implemented and enabled. The default key store is in-memory/dev-only — provision a real KMS for production ([deployment-security-checklist.md](deployment-security-checklist.md)).

## 6. Conventions

- **C#**: `net10.0`, nullable enabled, implicit usings, file-scoped namespaces, `sealed record` for contracts. Warnings are errors.
- **Commands/events** are immutable records in `Contracts`; **never** add infra dependencies to `Contracts` (architectural fitness test enforces this).
- **Event contract evolves additively** — new optional fields and new event types only; existing fields are never removed/renamed (forward-compatible subscribers). See [event-handler-patterns.md](event-handler-patterns.md).
- **Privacy**: never log event payloads or PII; projection logs carry only projection name + counts. Mark personal-data properties with `[PersonalData]`.
- **Idempotency**: command handlers are state-driven (`DomainResult.NoOp`); subscribers must dedupe and return `200 OK` for everything.
- **Versioning**: MinVer from git tags (prefix `v`); pre-release default `preview.0`.

## 7. Common tasks

| Task | Where |
|------|-------|
| Add a command + handler | record in `Contracts/Commands/`, validator in `Hexalith.Parties/Validation/`, `Handle(...)` in `Server/Aggregates/PartyAggregate.cs`, client method in `Client/`, optional MCP tool in `Mcp/Tools/` |
| Add an event | record in `Contracts/Events/` (implement `IEventPayload`), add `Apply(...)` to `PartyState`, update projection `Handlers/`, document in [event-handler-patterns.md](event-handler-patterns.md) |
| Add a read-model field | extend `PartyDetail`/`PartyIndexEntry`, update projection handler, surface via query/client |
| Add an MCP tool | `Mcp/Tools/PartiesMcpTools.cs` + `PartiesMcpToolNames.cs` |
| Change local DAPR/topology | `src/Hexalith.Parties.AppHost/` (`Program.cs` + `DaprComponents/`) |
| Deploy to K8s | `pwsh deploy/k8s/publish.ps1 -ConfirmContext <ctx>` ([deployment-guide.md](deployment-guide.md)) |
