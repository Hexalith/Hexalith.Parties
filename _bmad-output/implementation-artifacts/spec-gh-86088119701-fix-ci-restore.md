## Intent

**Problem:** GitHub Actions run `29009150186`, job `86088119701`, fails in the shared `ci / build-and-test` `Restore` step because a fresh checkout restores `Hexalith.Commons.Http` from `nuget.org`, where that package is not available. The new shared `domain-ci.yml` path initializes the root Commons submodule correctly, but the repo's default MSBuild routing keeps `Hexalith.Commons.Http` in package mode.

**Approach:** Add a narrow repo-local source fallback for `Hexalith.Commons.Http` when the root-declared `references/Hexalith.Commons` submodule is present, without switching all Hexalith dependencies to source mode. Add CI guard coverage so future workflow/release cleanup cannot reintroduce a fresh-checkout restore path that depends on the unpublished package.

## Boundaries & Constraints

**Always:** Keep `.github/workflows/ci.yml` and `.github/workflows/release.yml` delegated to `Hexalith.Builds` reusable workflows; keep root-declared submodule initialization only; preserve `.slnx` restore/build; keep `TreatWarningsAsErrors` and the warning-override guard intact; preserve package-only consumer validation and avoid leaking `ProjectReference` into generated consumer smoke projects.

**Ask First:** Changing `references/Hexalith.Builds`, publishing/bumping external Hexalith packages, disabling `run-consumer-validation`, or moving CI to full source mode for all Hexalith dependencies requires human approval.

**Never:** Do not use recursive submodule initialization, do not add `Version` attributes to project files, do not suppress NU1101 by weakening restore/build gates, do not remove package validation/tests, and do not add a private package feed or secret to mask this specific restore failure.

## Spec Change Log

## Design Notes

Do not repurpose `HexalithCommonsFromSource` for this fix. That switch currently controls `Hexalith.Commons`, `Hexalith.Commons.Http`, and `Hexalith.Commons.UniqueIds` together; setting it true whenever the Commons submodule exists would silently move more dependencies to source mode than the failing CI evidence requires. A dedicated `HexalithCommonsHttpFromSource` property keeps the fix explainable and reviewable.
