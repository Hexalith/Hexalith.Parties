## What Didn't Go Well (systemic patterns, no blame)

### 1. Cross-submodule fixes are not atomic with package-boundary proofs

**Story affected:** 3.1 (and inherited risk for 3.2)

Story 3.1's `PackedEventStoreContractsPackage_DoesNotLeakDaprOrInfrastructureDependencies` test depends on `Dapr.Actors PrivateAssets="all"` set inside the `Hexalith.EventStore` submodule. At review time, `git submodule status` reported the submodule as `-dirty`: the fix lived only in the working tree, so a fresh clone or CI build would produce a different `.nuspec` and fail the new test. Resolved by committing inside the submodule + bumping the pointer in this repo, but the gap between "story passes locally" and "story passes on clean clone" was discovered in review, not by tooling. Root cause: no pre-merge check enforces submodule cleanliness for stories whose evidence touches submodule contents.

### 2. Architecture pivots silently rotted Epic 3 planning text

**Stories affected:** 3.3, 3.4, 3.5

The original epic text (lines 1183–1289) named direct `/api/v1/parties` REST endpoints. Story 12.5 (predecessor) had already retired that surface and pivoted to the EventStore-fronted gateway. Each of 3.3, 3.4, 3.5 had to add an explicit `## Architecture Reconciliation` section to bridge the stale epic text to the accepted boundary. No epic text was ever updated to reflect the pivot. Same pattern would recur for any future epic authored before — and not swept after — a downstream architecture decision.

### 3. Solution-level build was broken throughout the epic

**Stories affected:** 3.1 (Debug Log), 3.2 (Debug Log), 3.10 (workaround)

`dotnet build .\Hexalith.Parties.slnx --configuration Release` failed across multiple stories on two out-of-scope blockers: (a) `CA2007` warnings-as-errors in `tests/Hexalith.Parties.DeployValidation.Tests/K8sStory93LintTests.cs` (resolved in 3.9), and (b) missing nested `Hexalith.Memories/Hexalith.Commons` submodule (sidestepped in 3.6 by making Memories opt-in). Story 3.10 then needed `-p:TreatWarningsAsErrors=false -p:WarningsAsErrors=` overrides because the `Hexalith.EventStore` submodule emits unrelated nullable/analyzer warnings promoted to errors. **No story owned "make the solution build green"; each story routed around the broken build with focused project tests.** Carried forward as TD-3-A.

### 4. Read-only sandbox blocked nested Codex sessions on four stories

**Stories affected:** 3.5, 3.6, 3.7, 3.8

Nested create-story Codex sessions were blocked from writing files and executing tests by read-only/no-approval child policy. All four stories required a "parent-managed Codex recovery flow" to author the story artifact and run validation. This is an automation/permissions gap, not a code issue, but it absorbed real cycles and is the proximate cause of Story 3.8's incomplete validation evidence (see Status Mismatch).

### 5. EventStore.Contracts public surface still references `Dapr.Actors.IActor`

**Story affected:** 3.1 (HIGH review finding, partially resolved)

`IProjectionActor` in `Hexalith.EventStore.Contracts` inherits from `Dapr.Actors.IActor`. The `PrivateAssets="all"` fix hides the package-level transitive flow (so Parties.Contracts clean-consumer tests pass) but the IL metadata reference to `Dapr.Actors` remains. Any downstream consumer that touches `IProjectionActor` will hit a `FileNotFoundException` for `Dapr.Actors.dll` unless they add a direct reference. Documentation action item still open ([HIGH] in 3.1 review). Real fix is to split `IProjectionActor` into a separate `Hexalith.EventStore.Contracts.Actors` package — cross-repo work in `Hexalith.EventStore`.

### 6. Story 3.8 closed without validation evidence

See **Status Mismatch** above. The story file says `in-progress` and `Pending validation`; sprint-status says `done`. Pattern risk: status was advanced based on artifact creation rather than green-test evidence. Captured as B5.

## Significant Discoveries (transferable design principles)

**Epic update required: NO** (Epic 4 already done; Epic 7 contracts unaffected). The discoveries below are transferable principles, not amendments.

1. **Cross-submodule edits must commit + bump the parent pointer atomically with the proof.** Working-tree-only submodule edits silently invalidate every downstream package-boundary proof because CI / a clean clone reproduces a different artifact. Any story whose evidence touches submodule contents must include "submodule clean + pointer bumped" as an acceptance check before review can approve. Captured as B4.
2. **Architecture pivots silently rot downstream epic story text.** Story 12.5 (EventStore-fronted client pivot) retired `/api/v1/parties` but Epic 3's planning text was never swept. Three stories (3.3, 3.4, 3.5) each had to author an `Architecture Reconciliation` section as a workaround. Future architecture pivots must include a sweep step: identify all epics referencing the retired surface and either update the text or tag the section as pivot-affected. Captured as B6.

## Key Takeaways (carry forward)

1. **EventStore-fronted gateway is the public boundary, full stop.** `src/Hexalith.Parties` is actor-host only — no public REST, OpenAPI, Swagger, MCP, or controllers, ever. The `ArchitecturalFitnessTests.PartiesAssembly_*` + `Program_SourceContainsOnlyActorHostMappingsAndDocumentedDaprInternalExceptions` + `PartiesProject_RemovesRestOpenApiAndMcpHostPackages` triple-lock is the discipline; do not weaken it.
2. **Package-boundary proof must inspect packed artifacts, not source builds.** Story 3.1's clean-local-feed consumer test is the template: `.nuspec` dependency graph, packed `lib/net10.0` assets, XML docs present beside the assembly, `[PersonalData]` discoverable from the *packaged* assembly. Source-tree project references can pass while external NuGet consumers still inherit forbidden dependencies.
3. **Senior Developer Review (AI) is a third pass that catches what design-review passes miss.** Party-mode and advanced elicitation catch design risk; Senior Dev Review (AI) caught low-level correctness bugs (JSON parsing guards, dead switches, missing standard HTTP headers, per-method test coverage gaps) on 3.1, 3.2, and 3.4. Use it on cross-cutting boundary stories. Captured as B7.
4. **Cross-submodule edits are not atomic with downstream proofs.** Always commit inside the submodule + bump the parent pointer in the same story that depends on the change, and verify on a clean clone. Captured as B4.
5. **Architecture pivots rot downstream epic text.** When a pivot retires a public surface, sweep all epics referencing it and either rewrite or tag the affected sections. Three Epic 3 stories paid an "Architecture Reconciliation" tax for an un-swept pivot. Captured as B6.
6. **Sprint-status `done` is a claim that must be backed by captured validation evidence in the story file.** Story 3.8 advanced to `done` without evidence; reconcile before claiming Epic 3 closed for audit (B5). Going forward, status-advance hooks should refuse to mark `done` until the Dev Agent Record contains a `dotnet test` (or equivalent) result line.
