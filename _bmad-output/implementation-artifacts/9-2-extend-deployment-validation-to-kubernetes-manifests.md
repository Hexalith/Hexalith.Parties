# Story 9.2: Extend Deployment Validation to Kubernetes Manifests

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As an operator preparing Parties for deployment,
I want deployment validation tooling to lint the aspirate-generated Kubernetes manifests in `deploy/k8s/` alongside runtime DAPR configuration,
so that unsafe or drifted K8s artifacts (missing probes, missing resource limits, dropped DAPR annotations, plaintext secrets, cloud-only capabilities, drifted DAPR Component/Configuration CRs) are caught **before** they reach a cluster ΓÇö using the same `validate-deployment.ps1` entry point Story 8.1 established.

## Acceptance Criteria

1. **K8s manifest workload lint catches missing image refs, DAPR annotations, ConfigMaps, and unresolved kustomization references (FR61) ΓÇö missing probes / resources / `:latest` tag are warnings in MVP**
   - Given `deploy/k8s/<app-id>/deployment.yaml` and `service.yaml` files generated by aspirate plus the existing regen.ps1 patches (the canonical per-app folder layout from Story 9.1, **without** any new manifest-shape changes from this story),
   - When the deployment validation tool runs in K8s-manifest mode against `deploy/k8s/`,
   - Then it reports as **blocking validation failures** (severity `fail`, exit code `1`): (a) any Deployment whose container image is missing, empty, or uses the placeholder `[]`/`null`; (b) any DAPR-enabled app id (`eventstore`, `eventstore-admin`, `parties`, `tenants`) whose Deployment is missing `dapr.io/enabled`, `dapr.io/app-id`, `dapr.io/app-port`, or `dapr.io/config` on **either** `metadata.annotations` **or** `spec.template.metadata.annotations`; (c) any Deployment whose `envFrom.configMapRef.name` references a ConfigMap that is not defined by any `kustomization.yaml`'s `configMapGenerator` under `deploy/k8s/`; (d) any entry in the top-level `deploy/k8s/kustomization.yaml` `resources:` list that does not resolve to an existing file or folder under `deploy/k8s/` (closes Story 9.1 deferred item).
   - And it reports as **non-blocking warnings** (severity `warn`, exit code stays `0` for these alone): (e) any Deployment whose container is missing `resources.requests` or `resources.limits` for `cpu` and `memory`, or where any limit is `0` / negative; (f) any Deployment missing both `readinessProbe` and `livenessProbe`; (g) any Deployment using an image tag of `:latest` (category `K8sWorkload-LatestImageTag`).
   - And app ids known to be **not** DAPR-enabled (`eventstore-admin-ui`, `parties-mcp`) do not trigger the DAPR-annotation check (the lint reads the app id from the workload metadata and uses the **same** DAPR-enabled-app-id set + the **explicit** non-DAPR exclusion set, both defined as constants in the script and mirroring `K8sManifestGenerationTests.daprAppToConfig`).
   - And hardening the aspirate output so the `warn` findings (e/f/g) clear is **explicitly deferred** to a follow-up story (see Deferred Decisions ΓåÆ "Aspirate output hardening (probes, resources, immutable tags)"). 9.2 is the *detection* surface, not the patch surface.

2. **DAPR Component / Configuration / Subscription drift lint against authoritative `deploy/dapr/` (FR61, FR39, FR40, FR41)**
   - Given `deploy/dapr/*.yaml` is the authoritative DAPR component source (Story 9.1 Completion Notes #4 + README parity table), and aspirate's broken `dapr/statestore.yaml` / `dapr/pubsub.yaml` placeholders are stripped by `regen.ps1`,
   - When the validation tool inspects the DAPR surface,
   - Then it reports as blocking validation failures: (a) the regen invariant violated ΓÇö `deploy/k8s/dapr/statestore.yaml` or `deploy/k8s/dapr/pubsub.yaml` exists (i.e. regen.ps1 did not strip them) **or** the top-level `deploy/k8s/kustomization.yaml` references them; (b) any `Configuration` CR under `deploy/dapr/accesscontrol*.yaml` whose `spec.accessControl.defaultAction` is **not** `deny`, OR whose policy list contains a wildcard `appId: '*'` / `appId: '**'`, OR whose operation paths use a wildcard `*` outside the documented `/**` form, OR which is missing one of the three required Parties operation rules (`eventstore-admin`, `tenants`, `parties` for `accesscontrol.yaml`; the locked-down empty-policies shape for `accesscontrol.eventstore-admin.yaml`; the per-service shape for `accesscontrol.parties.yaml` and `accesscontrol.tenants.yaml`); (c) any `Subscription` CR under `deploy/dapr/subscription-*.yaml` whose `pubsubname` does not equal `pubsub`, or whose `deadLetterTopic` is missing/empty, or whose route shape no longer matches the documented per-tenant routing pattern; (d) any `deploy/dapr/*.yaml` file present at story 9.1 close (the authoritative file list in `K8sManifestGenerationTests.AuthoritativeDaprTemplatesRemainTheBackingComponentSource`) that is missing.
   - And the report identifies the unsafe / drifted configuration **category** (e.g. `DAPR-ACL-DefaultActionNotDeny`, `DAPR-ACL-WildcardAppId`, `DAPR-Subscription-MissingDeadLetter`, `DAPR-Regen-PlaceholderNotStripped`) **without** echoing the raw YAML body, env-var values, tenant identifiers, or secret references in the user-facing output.

3. **Plaintext-secret / embedded-credential / hard-coded tenant-id lint in K8s manifests, with explicit allow/deny lists (FR61)**
   - **Scope of the scan:** the validation tool inspects ONLY `Deployment.spec.template.spec.containers[*].env`, every `configMapGenerator.literals` block under any `deploy/k8s/**/kustomization.yaml`, every `Secret` resource's `data`/`stringData`, every `envFrom`/`valueFrom` reference, and YAML anchor definitions (`&anchor value`). The scan **does NOT** inspect: YAML annotations (`metadata.annotations` / `spec.template.metadata.annotations`), YAML comments (`# ...` lines), `metadata.labels`, `selector.matchLabels`, or anchor references (`*anchor` ΓÇö only the definition is the secret carrier). This carve-out is what prevents `dapr.io/enable-api-logging: 'true'` (annotation) and `# Bearer ...` (comment) from false-failing.
   - **Multi-document YAML handling:** every `*.yaml`/`*.yml` file is parsed as a multi-document stream (`---` separators). All documents are scanned, not just the first. A `kind: Secret` smuggled as the second document in a `Deployment + Secret` stream is detected.
   - When it detects (each match emits its own per-regex category code):
     - **(a-1) Plaintext credential** (`K8sSecret-PlaintextCredential`): generic key=value secret shape ΓÇö regex `(password|pwd|passwd|secret|token|api[_-]?key|client[_-]?secret)\s*[=:]\s*[^{<$\s]` (case-insensitive).
     - **(a-2) URL-embedded credential** (`K8sSecret-UrlEmbeddedCred`): `(?i)[a-z]+://[^/\s:@]+:[^/\s@]+@` ΓÇö Postgres/Redis/Mongo `proto://user:pw@host`.
     - **(a-3) JWT token literal** (`K8sSecret-JwtTokenLiteral`): `eyJ[A-Za-z0-9_\-]{10,}\.[A-Za-z0-9_\-]{10,}\.[A-Za-z0-9_\-]{10,}`.
     - **(a-4) AWS access key** (`K8sSecret-AwsAccessKey`): `AKIA[0-9A-Z]{16}`.
     - **(a-5) Azure connection string** (`K8sSecret-AzureConnString`): `DefaultEndpointsProtocol=https;`.
     - **(a-6) Private key PEM header** (`K8sSecret-PrivateKey`): `-----BEGIN [A-Z ]+ PRIVATE KEY-----`.
     - **(b) Static tenant id** (`K8sSecret-StaticTenantId`): any env var or literal whose **key** matches `^(Tenants__TenantId|TENANT_ID|DEFAULT_TENANT|.*__TenantId)$` with a value that is anything other than `{env:*}` / `valueFrom` / empty / placeholder.
     - **(c) Committed secret value** (`K8sSecret-CommittedSecretValue`): any `Secret` resource of `type: Opaque` whose `stringData` or `data` is committed with a non-placeholder value. (Committed tree currently contains zero `Secret` resources; the check is forward-looking and MUST NOT false-positive on the empty case.)
   - And the **key-allowlist** (these keys never trigger the value-side plaintext regex, even if the value would match): `services__*__http__*`, `Tenants__ServiceName`, `Tenants__PubSubName`, `Tenants__TopicName`, `Tenants__Enabled`, `Tenants__CommandApiAppId`, `EVENTSTORE_HTTP`, `TENANTS_HTTP`, `ASPNETCORE_URLS`, `ASPNETCORE_FORWARDEDHEADERS_ENABLED`, `HTTP_PORTS`, `OTEL_DOTNET_EXPERIMENTAL_OTLP_RETRY`, `dapr.io/enable-api-logging` (annotation key). This list is enumerated as a constant in the script and mirrors the entries actually present in the committed `deploy/k8s/parties/kustomization.yaml` `configMapGenerator.literals` block.
   - And the **value-allowlist (placeholder set)** ΓÇö case-**in**sensitive, whitespace-trimmed: `{env:*}`, `{env:*|*}` (DAPR-style with default), `$(VAR)` / `${VAR}` (K8s-native env-var refs), any `valueFrom.fieldRef` / `valueFrom.configMapKeyRef` / `valueFrom.secretKeyRef` reference, `<set-by-operator>`, `<placeholder>`, `REPLACE_ME`, `replace_me`, empty string, or a `data:` block whose base64-decoded value matches one of those placeholders. URL-encoded placeholder forms (`%24%7Benv%3A...`) must also be recognized.
   - Then they are reported as blocking validation failures (severity `fail`, exit code `1`) with the category code above,
   - And the report explains the expected secret-reference pattern (use `{env:VAR}` in DAPR Components, `valueFrom.secretKeyRef` in Deployments, `Secret` resources with placeholder values + operator-managed real values) **without printing the offending value or surrounding context** ΓÇö the user-facing output prints `<redacted:N chars at <relative-file-path>:<line-number>>` where N is the offending string length. The same redaction contract applies to the `recommendation` string (never interpolate the offending value into the recommendation).

4. **Local-cluster capability allowlist lint (FR61, NFR30)**
   - Given the validation tool is parameterized for the local-cluster MVP target (default mode) versus a future managed-cloud mode (explicit opt-in),
   - When a manifest in `deploy/k8s/` references a cloud-provider-only capability (a `StorageClass` named in a documented cloud-provider list ΓÇö `managed-csi`, `gp2`, `gp3`, `azurefile`, `standard-rwo`, `pd-standard`, `pd-ssd`, etc., OR an `IngressClass` named `alb`, `nginx-aws`, `gce`, `azure/application-gateway`, OR `loadBalancer.ingress` annotations that pin a cloud-provider scheme like `service.beta.kubernetes.io/aws-load-balancer-type`, `service.beta.kubernetes.io/azure-load-balancer-internal`, OR a `Service` of `type: LoadBalancer` without the documented local-cluster opt-out annotation),
   - Then the tool reports a blocking validation failure for the MVP scope with category `K8s-NonLocalClusterCapability` and a remediation pointer to the documented post-MVP escalation path in `deploy/k8s/README.md` ΓåÆ "Out of MVP scope".
   - And the lint is enabled by default; an explicit `-AllowCloudCapabilities` switch (or equivalent `--allow-cloud-capabilities`) is required to demote these failures to warnings (post-MVP path).

5. **Bounded, machine-readable, secret-safe, deterministically-ordered validation output (FR61, FR43, NFR security)**
   - Given the K8s-manifest lint runs in either `console` or `json` output mode (same `--output` switch Story 8.1 established for `validate-deployment.ps1`),
   - When failures, warnings, or successes are reported,
   - Then the output is **bounded** (per-finding details capped at a documented character budget ΓÇö e.g. Γëñ 500 chars per finding, Γëñ 50 findings per category before truncation with a `"N additional findings suppressed ΓÇö re-run with --output json for full list"` marker),
   - And the output is **machine-readable** in `--output json` mode (each finding includes `category`, `code`, `severity` Γêê `{fail, warn, pass}`, `target` ΓÇö relative file path + optional YAML pointer like `deployment.yaml#/spec/template/spec/containers/0`, `recommendation` ΓÇö human-readable remediation string, **NOT** a `value` field for the offending content),
   - And findings are emitted in a **deterministic order** (sorted ascending by `(category, code, target)` triple) so a second run against the unchanged tree produces byte-identical JSON modulo a single `timestamp` field. PowerShell hashtable enumeration order is **not** trusted; the script must sort explicitly before emission.
   - And recommendation strings are **parametrized constants per category code**. The interpolated values in any user-facing string (stdout, stderr, JSON `recommendation` field) are limited to: the bounded `N` (redacted length), the relative file path, the line number, and the category code. **Nothing else** ΓÇö no field name, no key name, no YAML path beyond the file:line, no `Exception.Message`, no `Exception.StackTrace`. A `recommendation: "Expected '{env:VAR}' but got '<value>'"` shape is **forbidden** because it interpolates `<value>`.
   - And file paths in the output are **sanitized for log injection**: any control character (`\r`, `\n`, `\t`, `\b`, `\x00`-`\x1f`) in a file path is replaced with `?` before emission. A maliciously-named file `evil\n[FAKE:fail]\n.yaml` cannot inject a fake finding line into the console output.
   - And the output **never** prints (to **stdout, stderr, or the exit-code message**): raw env-var values, `Secret.data`/`Secret.stringData` contents (decoded or encoded), ConfigMap `literals` containing tenant identifiers / connection strings / tokens, bearer/JWT tokens, DAPR access-control policy raw YAML, personal-data-marked fields, full file contents of any ConfigMap/Secret, or PowerShell stack traces (every `catch` block surfaces a bounded category-coded message, never the raw `$_.Exception.ToString()`).
   - And the exit code contract from Story 8.1 is preserved: `0` = all checks passed (warnings OK), `1` = at least one blocking failure, `2` = invalid arguments / config path not found. A new exit code `3` is **not** introduced (`deploy-local.ps1` reserves `3` for missing tooling; the validator stays in the `0`/`1`/`2` contract).

6. **Test coverage for valid manifests, every drift category, plaintext secrets, non-local capability, secret-leak entropy property, name-baseline regression guard, and backwards-compatibility (FR61)**
   - Given xUnit tests in `tests/Hexalith.Parties.DeployValidation.Tests/` (the project Story 8.1 created),
   - When they run inside `pwsh scripts/test.ps1 -Lane deploy`,
   - Then they cover **at minimum** (target ΓëÑ 25 new tests):

     - **(b) Missing-probe / missing-resource warnings.** Synthetic fixtures: no probes ΓåÆ `warn` `K8sWorkload-MissingProbes`; no `resources.requests` ΓåÆ `warn` `K8sWorkload-MissingResources`; both ΓåÆ both warnings, exit `0`.
     - **(c) Missing DAPR annotation per app id** (`eventstore`, `eventstore-admin`, `parties`, `tenants`) ΓåÆ blocking `K8sWorkload-MissingDaprAnnotation`. Plus a negative test: removing the annotation on `eventstore-admin-ui` or `parties-mcp` does **not** trigger the check (validates the exclusion-set constant).
     - **(d) Drifted DAPR ACL.** Three positive tests: `defaultAction: allow`, wildcard `appId: '*'`, missing per-service operation rule. Two negative tests: per-service `appId: eventstore-admin` allowlisted, documented `/**` operation path accepted.
     - **(e) Drifted subscription.** Missing `deadLetterTopic`, wrong `pubsubname` (not `pubsub`) ΓåÆ blocking `DAPR-Subscription-*`.
     - **(f) Plaintext-secret regex set.** **Each regex needs ΓëÑ 1 positive and ΓëÑ 2 negative tests.** Positives: `password=foo123`, `postgres://u:p@h`, JWT shape, `AKIA....`, PEM header, `DefaultEndpointsProtocol=https;...`. Negatives per regex: `{env:DB_PASSWORD}` (placeholder), `REPLACE_ME` (placeholder), `services__eventstore__http__0=http://eventstore:8080` (key-allowlist), `Tenants__PubSubName=pubsub` (key-allowlist), `dapr.io/enable-api-logging: 'true'` (key-allowlist), `retryAfterHeader: ...` (no key match), `Bearer ...` inside a YAML comment (`#` line), `AKIA` substring inside a Git SHA-like string. URL-encoded placeholder `%24%7Benv%3ADB_PASSWORD%7D` accepted.
     - **(g) Static tenant id.** `Tenants__TenantId=acme-corp` ΓåÆ blocking `K8sSecret-StaticTenantId`; `Tenants__TenantId={env:TENANT_ID}` ΓåÆ pass.
     - **(h) Committed secret value.** `kind: Secret` with `stringData.password: hunter2` ΓåÆ blocking `K8sSecret-CommittedSecretValue`; same with `stringData.password: REPLACE_ME` ΓåÆ pass; same with no `stringData`/`data` at all ΓåÆ pass.
     - **(i) Cloud-only capability** (MVP mode + opt-out switch).      - **(k) Output safety ΓÇö poisoned-string sweep.** Fixture with deliberately distinctive poison strings (`__POISONED_TENANT_DO_NOT_LEAK__`, `__POISONED_CONNSTR_DO_NOT_LEAK__`, etc.) in every literal slot.      - **(r) Backwards-compat Story 8.1 invariant.** Re-run `pwsh deploy/validate-deployment.ps1 --config-path deploy/dapr` (Story 8.1 mode, no `-K8sPath`) and assert exit `0` with the committed tree.
     - **(s) Unparseable YAML.** Fixture with malformed YAML ΓåÆ blocking `K8s-YamlParseError`; assert the offending file content is **not** echoed to stdout/stderr.
   - And the existing Story 8.1 / 3.9 DAPR-component-template validation tests (`DeploymentValidationTests`, `AppHostDaprTopologyValidationTests`, `TenantsDeploymentValidationTests`, plus the Story 9.1 fitness tests `K8sManifestGenerationTests`, `K8sLocalContextAllowlistTests`) **continue to pass unchanged** ΓÇö Story 9.2 is **additive** to those.

## Response Outcome Boundaries

| Scenario | Expected outcome |
|---|---|
| `deploy/k8s/` is empty or missing | Lint exits `2` with `"k8s-path not found"`; does not partial-run. |
| `deploy/dapr/` is empty or missing | Lint exits `2` with `"dapr-path not found"`; does not partial-run. |
| Single deployment.yaml present but malformed YAML | Lint exits `1` with category `K8s-YamlParseError` and the file path; does **not** print the unparseable content (bounded message only). |
| 100+ findings in a single category | Console output prints first 50, then `"N additional findings suppressed ΓÇö re-run with --output json for full list"`. JSON output returns all findings. |
| Deployment uses an image with a digest pin (e.g. `parties@sha256:...`) | Passes the image check; the digest counts as a present reference (digest pins are encouraged, not flagged). |
| Deployment uses image tag `:latest` (the committed default) | **Warning**, not failure ΓÇö `K8sWorkload-LatestImageTag` category, exit code stays `0`. Hardening to immutable tags is part of the deferred "Aspirate output hardening" story. |
| Deployment is missing probes / `resources.requests` / `resources.limits` (the committed default) | **Warning**, not failure ΓÇö `K8sWorkload-MissingProbes` / `K8sWorkload-MissingResources` categories, exit code stays `0`. Same deferred-hardening story. |
| ConfigMap value contains a string matching the secret-shape regex but the surrounding context is `{env:VAR}` reference syntax (the documented placeholder) | Passes ΓÇö the value-placeholder set explicitly excludes `{env:*}` references; tested directly. |
| ConfigMap key is on the key-allowlist (e.g. `Tenants__PubSubName=pubsub`) even though the value would match the regex | Passes ΓÇö key-allowlist is checked **before** the value regex fires; tested directly per key. |
| Operator runs with `-AllowCloudCapabilities` on a manifest set that also has missing DAPR annotations | Cloud-capability findings demote to warn; missing-DAPR-annotation findings stay blocking. The two lint dimensions are independent. |
| Symbolic-link inside `deploy/k8s/` pointing outside the repo | Lint refuses to follow the link (treated as a missing file ΓåÆ blocking `K8s-PathTraversal`); does not stat anything outside the repo root. |
| A new authoritative `deploy/dapr/<new-component>.yaml` is added between Story 9.1 close and 9.2 implementation | Validation must read the file list from `deploy/dapr/` directly, not a hardcoded list, OR maintain the hardcoded list in **one** place that `K8sManifestGenerationTests.AuthoritativeDaprTemplatesRemainTheBackingComponentSource` already enforces (preferred ΓÇö extend the existing list, don't fork it). |
| Two consecutive lint runs against an unchanged tree | Byte-identical JSON output modulo a single `timestamp` field; findings sorted ascending by `(category, code, target)`. Asserted by Test (n) in AC6. |
| PowerShell `catch` block fires (malformed YAML, missing file, permission denied) | Bounded category-coded message to stdout; stderr remains empty or carries the **same** bounded message ΓÇö never the raw `$_.Exception.ToString()` and never the file contents that triggered the catch. |

## Out of Scope

The following are **explicitly NOT this story's job**. Dev must surface a scope-creep flag if any of these creep into a PR draft:

- **Modifying any artifact under `deploy/k8s/`.** `regen.ps1`, `*/deployment.yaml`, `*/service.yaml`, `*/kustomization.yaml`, top-level `kustomization.yaml`, `namespace.yaml`, `deploy-local.ps1`, `teardown-local.ps1`, `README.md` content beyond the new "K8s manifest lint" subsection ΓÇö all unchanged. 9.2 is detection-only.
- **Injecting probes / `resources` / immutable image tags into generated manifests.** Deferred to "Aspirate output hardening" follow-up (see Deferred Decisions).
- **Wiring the validator into `deploy-local.ps1` as a pre-flight.** Deferred to "Auto-validate pre-flight" follow-up.
- **Modifying the 8.1 / 11.4 / 9.1 check categories or test files.** `DeploymentValidationTests`, `AppHostDaprTopologyValidationTests`, `TenantsDeploymentValidationTests`, `K8sManifestGenerationTests`, `K8sLocalContextAllowlistTests` ΓÇö all stay green unchanged.
- **Managed-cloud lint rules** (AKS/EKS/GKE-specific assertions, ingress-controller wiring, TLS issuers, cloud storage classes). AC4 is **detective only** (don't ship cloud-only resources accidentally), not **prescriptive** (here's how to configure for AKS).
- **Auto-fix / patch-mode for findings.** Validator is read-only.
- **YAML library dependency** (`powershell-yaml` / `ConvertFrom-Yaml`). Use the hand-rolled regex helpers already in `validate-deployment.ps1`.
- **Cross-service env-var symmetry lint** (parties-env vs tenants-env). Operator config asymmetry is a follow-up.
- **Aspirate `dapr.io/enable-api-logging` / `ASPNETCORE_FORWARDEDHEADERS_ENABLED` hardening.** Both are aspirate defaults; addressing them requires the kustomize-overlay layer that's out of MVP.
- **New `Hexalith.Parties.*` C# project.** Tests go in the existing `Hexalith.Parties.DeployValidation.Tests` project.
- **Sibling-script Option B** (creating `deploy/validate-k8s-manifests.ps1`). Option A decided; Option B requires reviewer escalation, not silent choice.

Only `configMapGenerator.literals`, container `env`, `envFrom`/`valueFrom`, `Secret.data`/`Secret.stringData`, and YAML anchor definitions (`&anchor value`) are in scope. Anchors that are *referenced* (`*anchor`) are not the secret carrier ΓÇö only the definition is.
- Do **not** parse YAML files as single-document. All `*.yaml`/`*.yml` files in `deploy/k8s/` and `deploy/dapr/` are read as multi-document streams (`---`-separated) and every document is scanned. Use the sentinel prefix `~ENT~` followed by 28 random alphanumeric chars; the `~` character is not used in any category code, file path, or recommendation string.
