### Foundation (UI system — both spines inherit)

- **UI system:** **Hexalith.FrontComposer** (shell/host) + **FluentUI Blazor V5**
  (`Microsoft.FluentUI.AspNetCore.Components` 5.0.0-rc.3) component library.
  → `DESIGN.md` tokens reference/extend FluentUI V5 + FrontComposer defaults;
  `EXPERIENCE.md` specifies only the behavioral delta on top of the system.
- **Surfaces in scope (two portals, shared UI system):**
  1. **Parties Admin portal** — internal admin of person/organization records.
  2. **Parties Consumer portal** — consumer-facing (new vision; not yet in repo).
- **Hosting / deployment:** both portals served by a web service named
  **`parties-ui`**. (New service name — distinct from the actor host `parties`,

  responsive incl. mobile web; confirm in journeys).

### Product shape

- **One app, role-gated.** A single `parties-ui` Blazor app with two role-gated
  areas — **Admin** and **Consumer** — under one roof, same UI family.
- **Consumer's job = self-service over one's *own* data** (confirmed): view own
  profile, edit it, manage consent, request erasure (GDPR self-service). The
  crypto-shredding/erasure machinery already exists server-side; this is its
  human face.
- **Admin's job** (existing direction): manage person/organization party records
  across tenants.
- **Feel:** *simple to use* with a **FluentUI "enterprise" look & feel** —
  clean, professional, Microsoft-Fluent-native. Admin and Consumer read as one
  family (single app), not two brands.

### Decisions

- **Stakes: Consumer-facing.** → brand/trust/onboarding/polish carry weight;
  accessibility floor targets WCAG AA+; accessibility reviewer lens recommended
  at the Reviewer Gate. **Tension to resolve:** repo today is internal-admin +
  developer-picker + MCP, so a consumer-facing vision implies surface(s) beyond
  what's built. Probe in brain dump.

### Grounding captured (2026-06-09, two research subagents)

- **Inheritance backbone:** FrontComposer Shell (`FrontComposerShell.razor`) = FluentUI V5
  `<FluentLayout>` Header(48px)/Nav(220px·48px rail)/Content(`Padding.All3`)/Footer +
  `<FluentProviders/>`. Provides theme toggle (Light/Dark/**System** via `IThemeService`),
  command palette (Ctrl+K), settings (Ctrl+,), skip links, density (`--fc-spacing-unit`
  2/4/6px), nav auto-populated from domain manifests, semantic aliases `--fc-color-*`.
  **Accent default `#0097A7` (teal)** → `--accent-base-color` → `ThemeSettings.accentBaseColor`.
- **Token family to cite:** Fluent 2 — `--colorNeutral{Foreground1..4,Background1..6,Stroke1..3}`,
  `--colorBrand{Background,Foreground1/2,Stroke1/2}`, `--colorStatus{Success,Warning,Danger,Info}Foreground1`,
  `--colorStrokeFocus2`, `--fontFamilyBase` (Segoe UI), `--fontSizeBase100..600`/Hero700..1000,
  `--fontWeight{Regular,Medium,Semibold,Bold}`, `--spacing{Horizontal,Vertical}{XXS..XXXL}`,
  `--borderRadius{Small2,Medium4,Large6,XLarge8,Circular}`, `--shadow2..64`, `--strokeWidth{Thin,Thick}`.
- **AdminPortal today** (`PartiesAdminPortal.razor`): master-detail; `FluentDataGrid`,
  `FluentTextInput`(search), `FluentSelect`(type/state filters), `FluentButton`
  (`ButtonAppearance.Primary/Outline/Subtle`). Routes `/admin/parties`, `/admin/parties/{id}`,
  `/admin/parties/{id}/gdpr`. Rich state machine `StatusKind` {Loading, Loaded, DisplayNameOnly,
  Degraded, SignInRequired, TenantUnavailable, AdminRequired, Forbidden, TransientFailure, NoData,
  LoadFailure, Validation}. Strong a11y already (aria-live, focus mgmt). GDPR panels exist
  (erasure/consent/portability/restriction/processing-records/verification-report/DPO-summary).
- **Picker** (`<hexalith-party-picker>`): combobox; dispatches `party-selected`
  `{partyId, partyType, status}` (`composed:true`); state machine `PartyPickerSearchState`
  {Idle,Loading,Ready,Empty,LocalOnly,Degraded,Unauthorized,Forbidden,TransientFailure,NotFound,Gone,Error};
  300ms debounce. **Flag:** styled with legacy FAST tokens (`--neutral-stroke-rest`, `--accent-fill-rest`)
  that won't resolve against the v5 Fluent 2 shell — needs modernizing (logged as design debt).
- **Net-new (don't exist yet):** consumer self-service area, a tenant switcher, a header user-chip.

**Resolutions applied this run (critical + high + key mediums):**
1. **Contrast:** DESIGN.md — raw accent `#0097A7` is non-text only (3.51:1);
   filled buttons bind to `--colorBrandBackground` (AA, ≈#00767f); added
   `brand-fill` token + Do/Don't rows; false "ratios hold" claim removed. All 5
   mocks repointed to `--brand-fill:#00767f`.
2. **Live-region politeness split:** polite for status/freshness/processing,
   `role=alert` (assertive) for validation-rejection/failure.
3. **Combobox ARIA:** full WAI-ARIA combobox mandated on the picker (folded into
   the re-skin debt); create-edit mock wired with role=combobox/listbox/option +
   aria-activedescendant/selected.
4. **Semantic controls:** consent = `role=switch`/`aria-checked`; type chooser =
   `radiogroup`; erase confirm = real labeled `<input>`. Spine forbids interactive
   `<div>`s; mocks updated.
5. **Focus contract:** per-surface — trap/restore on dialogs, move-to-alert on
   blocking errors, announce-only (no focus steal) on optimistic saves.
6. **Erasure copy:** commit to *start* not finish; state both halves
   (cancellable-until-begins + permanent-once-done); new "Erasure requested / in
   progress" state row; neutral (not success-green) ack.
7. **Lawful-basis honesty:** "Things you control" (consent, default **Off**, Art. 7)
   vs "Things we keep to run your account" (contract / legitimate interest, read-only
   + **Object** Art. 21). Added "no consent dark patterns" anti-pattern.
8. **Export expectation:** dropped sub-minute promise; machine-readable (JSON);
   "we'll show it when it's ready." Fixed profile email view↔edit drift +
   Saved/Saving conflict; anchored "Manage all consent".

**Residuals (medium/low, documented, non-blocking):** phone-reflow mock for the
Admin master-detail deferred; sub-24px decorative controls + 12px secondary text in
mocks are illustrative (real build floors consumer secondary text 13–14px, 44px
touch slop); picker re-skin (FAST→Fluent 2 tokens + ARIA) carried as design debt;
PII-in-copy spec is clean — implementation must keep the admin typed-name in-memory.
